Key Concepts: JDK, JRE, and JVM

JDK (Java Development Kit): Used by developers. It includes tools (like compiler javac) to write, compile, and build Java applications.
JRE (Java Runtime Environment): Part of JDK. It provides the libraries and JVM needed to run Java programs (but no development tools).
JVM (Java Virtual Machine): The core engine that runs Java programs. It converts bytecode (.class files) into machine code that the computer understands.

Important Points:

Java is platform-independent ("Write Once, Run Anywhere") because the bytecode is the same everywhere.
But JVM is platform-dependent â€” each operating system (Windows, Linux, macOS) has its own version of JVM to handle hardware differences.
Java is called a compiler-interpreter language (hybrid):
First, the compiler (javac) turns source code (.java) into bytecode.
Then, the JVM interprets or uses JIT (Just-In-Time) compiler to run the bytecode line by line or in optimized blocks.


This makes Java secure, portable, and efficient!
